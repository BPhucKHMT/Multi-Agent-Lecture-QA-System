## Streaming Scenario Deep Dive

### Why Streaming Breaks JSON Parsing:

The system uses `workflow.astream_events()` which:
1. Calls LLM with streaming=True (llm_model.py line 17)
2. Receives tokens one by one via astream_events()
3. Each token is a PARTIAL JSON fragment
4. JsonOutputParser.parse() tries to parse INCOMPLETE JSON
5. Fails immediately (can't parse `{"quizzes": [{"question"` - incomplete)
6. Fallback parser called on partial JSON
7. Returns None (can't extract valid JSON from fragment)
8. Exception re-raised → OUTPUT_PARSING_FAILURE

# tests/rag_core/test_quiz_json_parsing_streaming.py

import pytest
import json
from src.rag_core.agents.quiz import _extract_quiz_json_payload, QuizOutput

class TestStreamingIncompleteJSON:
    """Test suite for streaming incomplete JSON edge cases"""
    
    def test_streaming_incomplete_missing_closing(self):
        """Streaming stops before final closing braces"""
        raw = '{"quizzes": [{"question": "Q1", "options": ["A"'
        result = _extract_quiz_json_payload(raw)
        assert result is None, "Should return None for incomplete JSON"
    
    def test_streaming_incomplete_inside_string(self):
        """Streaming stops inside a string value"""
        raw = '{"quizzes": [{"question": "Incomplete question'
        result = _extract_quiz_json_payload(raw)
        assert result is None
    
    def test_streaming_incomplete_after_first_question(self):
        """Streaming provides complete first question but incomplete wrapper"""
        raw = '{"quizzes": [{"question": "Q1", "options": ["A","B","C","D"], "correct_answer": "A", "explanation": "E", "video_url": "url", "timestamp": "00:00:01"}'
        result = _extract_quiz_json_payload(raw)
        assert result is None, "Should not parse incomplete wrapper"


class TestJSONSyntaxErrors:
    """Test JSON syntax errors LLM might produce"""
    
    def test_trailing_comma_in_array(self):

        result = _extract_quiz_json_payload(raw)
        assert result is None
    
    def test_multiple_json_objects(self):
        """Text contains multiple JSON objects"""

## Success Criteria

A test is considered PASSING if:
1. Incomplete JSON returns None (not exception)
2. Syntax errors return None (not exception)
3. Validation failures are caught by Pydantic (not silently accepted)
4. Streaming scenarios don't break the entire response chain
5. Error messages are clear enough to debug LLM behavior

- Streaming JSON incompleteness handled gracefully
- Type validation errors detected early
- Fallback parser robust to 95%+ of real LLM outputs
- User sees meaningful error messages instead of generic OUTPUT_PARSING_FAILURE
