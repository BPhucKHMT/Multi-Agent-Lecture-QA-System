2. MISSING/FRAGILE SHAPES (ROOT CAUSES)
────────────────────────────────────────────────────────────────────────────────

🔴 CRITICAL - STREAMING INCOMPLETE JSON:
   Problem: JsonOutputParser receives partial JSON from astream_events()
   Example: {"quizzes": [{"question": "Q1"  (no closing braces)
   Current: Throws exception → Re-raised as OUTPUT_PARSING_FAILURE
   Impact: All streaming quiz generation fails silently
   
   Root: LLM generates via streaming=True, JSON comes in chunks
         JsonOutputParser expects complete JSON
         Fallback parser can't fix fragments
         No recovery → user sees generic error

⚠️ HIGH - UNESCAPED NEWLINES IN JSON VALUES:
   Problem: LLM embeds literal \n in string field (not escaped)
   Example: {"question": "Line1\nLine2"}  (actual newline, not \n escape)
   Current: json.loads() fails silently → returns None
   Impact: Legitimate responses rejected

⚠️ HIGH - TRAILING COMMA IN JSON:
   Problem: LLM adds trailing comma (rare but happens)
   Example: {"quizzes": [{...},]}
   Current: json.loads() fails → returns None
   Impact: Otherwise valid response rejected

⚠️ MEDIUM - TYPE MISMATCHES:
   Problem: LLM generates wrong types for fields
   Examples:
     - options: "ABCD" (string instead of ["A","B","C","D"])
     - timestamp: 120 (number instead of "00:02:00")
   Current: JSON parse OK → Pydantic validation fails silently
   Impact: No clear error message about what field is wrong

⚠️ MEDIUM - ARRAY LENGTH MISMATCH (Tutor):
   Problem: Tutor returns arrays with different lengths
   Example: video_url=[2 items], filename=[1 item]
   Current: JSON parse OK → No validation of array alignment
   Impact: Frontend crashes when accessing by index

⚠️ LOW - EMPTY REQUIRED FIELDS:
   Problem: Empty quizzes array or missing quizzes key
   Current: JSON parse OK → Pydantic rejects but error buried
   Impact: Unclear what went wrong

3. FALLBACK PARSER ASSESSMENT
────────────────────────────────────────────────────────────────────────────────

Function: _extract_quiz_json_payload() (quiz.py lines 43-64)

✅ Strengths:
   - Handles fenced JSON with markdown
   - Handles text-prefixed JSON
   - Regex is case-insensitive
   - Gracefully returns None on parse failure

❌ Weaknesses:
   - GREEDY REGEX: (\{[\s\S]*\}) matches first { to LAST }
     * Problem: If text contains "A {...} B {...} C", extracts A through C
     * Can produce invalid JSON if multiple objects in text
   
   - NO VALIDATION AFTER EXTRACTION:
     * Accepts {"quizzes": []} (empty but valid JSON)
     * Accepts malformed but-parseable JSON
     * Pydantic catches errors but information is lost
   
   - SILENT EXCEPTIONS:
     * No logging of parse failures
     * Makes streaming debugging very difficult
     * Users see generic "OUTPUT_PARSING_FAILURE"
   
   - INCOMPLETE JSON NOT RECOVERED:
     * Design assumes fragments are just "missing wrapper text"
     * Doesn't handle streaming chunks that are mid-field
     * Example: {"quizzes": [{"question": "Incomplete → Returns None

4. STREAMING SCENARIO ROOT CAUSE (HIGHEST CONFIDENCE)
────────────────────────────────────────────────────────────────────────────────

THE PROBLEM:
When user generates quiz in streaming mode, error rate is high.
Error: "Xin lỗi, có lỗi xảy ra" with type="error"
Root cause: OUTPUT_PARSING_FAILURE in fallback

THE FLOW:
1. User requests: "Tạo quiz về CNN"
2. node_quiz() called → prompt | llm | parser chain invoked
3. LLM starts streaming tokens via astream_events()
   - Chunk 1: {"quizz
   - Chunk 2: es": [{"quest
   - Chunk 3: ion": "What is CNN?", "options": ["A","B","C",
   - ...
4. JsonOutputParser.parse() receives INCOMPLETE JSON at some point
   - Tries: json.loads({"quizzes": [{"question"...)
   - Fails: JSONDecodeError - expected more data
5. Exception caught in quiz.py line 157
6. Fallback invoked: raw_result = llm_chain.invoke()
   - But this returns the PARTIAL content from streaming
   - Fallback parser called on PARTIAL JSON
   - Returns None (can't fix fragments)
7. Condition "if not repaired" is TRUE
8. Re-raises exception → OUTPUT_PARSING_FAILURE
9. User sees error response

WHY FALLBACK FAILS:
- _extract_quiz_json_payload() designed for text extraction
- Assumes fragments are like "Prefix {...}" (extractable)
- Doesn't handle "Incomplete partial object" (unrecoverable)
- No accumulation/buffering mechanism

WHY IT WORKS FOR SUPERVISOR:
- Supervisor uses streaming=False (llm_model.py line 35)
- Gets complete JSON in one response
- _normalize_tool_call_payload() succeeds
