SUMMARY TABLE: JSON PARSING EDGE CASES

════════════════════════════════════════════════════════════════════════════════

1. CURRENTLY HANDLED (by _extract_quiz_json_payload):
   ✅ Plain JSON: {"quizzes": [...]}
   ✅ Fenced JSON: ```json {...}```
   ✅ Text-prefixed: "Prefix\n{...}"
   ✅ JSON with newlines/Unicode
   ✅ Nested JSON in field values

2. CURRENTLY UNHANDLED - IMMEDIATE FAILURES:
   ❌ Incomplete JSON (streaming scenario)
       Status: Returns None → exception re-raised
       Example: {"quizzes": [{"question": "Q1"
   
   ❌ Trailing comma: {"quizzes": [{...},]}
       Status: json.loads() fails silently
   
   ❌ Unescaped newlines in string value
       Status: json.loads() fails silently

3. CURRENTLY UNHANDLED - SILENT PASSES THEN VALIDATION FAILS:
   ⚠️ Empty object: {}
       Parser: OK (JSON valid)
       Validator: FAILS (missing "quizzes" field)
   
   ⚠️ Wrong types: {"quizzes": [{..., "options": "ABCD"}]}
       Parser: OK (JSON valid, all keys present)
       Validator: FAILS (expects array not string)
   
   ⚠️ Array instead of object: [{...}]
       Parser: Matches as JSON, loads OK
       Validator: FAILS (expects {"quizzes": [...]})
   
   ⚠️ Wrong element count: {"quizzes": []} (valid but empty)
       Parser: OK → model_validate OK
       Result: Valid but useless output

4. SUPERVISOR TOOL PARSING - ROBUSTNESS ISSUES:
   ⚠️ Non-dict args: {"name": "X", "args": "string"}
       Result: Silently coerced to {} → loses query parameter
   
   ⚠️ tool_calls with multiple items: [{"name": "A"}, {"name": "B"}]
       Result: Only first item processed (queue loss)

5. TUTOR PARSING - ROBUSTNESS ISSUES:
   ⚠️ Array length mismatch: video_url=[2 items], filename=[1 item]
       Parser: OK (JSON valid)
       Result: Frontend crashes on index access

6. STREAMING SCENARIO - ROOT CAUSE:
   🔴 CRITICAL: Incomplete JSON during streaming
       Why: JsonOutputParser receives partial JSON from stream
       When: astream_events() yields chunks during LLM generation
       Result: Parser exception → Fallback gets partial JSON → Returns None → Error
       
       Flow:
       astream_events() [partial JSON] 
         → JsonOutputParser.parse() [fails on incomplete]
         → Fallback _extract_quiz_json_payload(partial_json)
         → Returns None (can't fix streaming fragment)
         → Re-raises exception
         → User sees OUTPUT_PARSING_FAILURE
